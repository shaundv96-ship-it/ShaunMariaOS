# ❤️ ShaunMariaOS Principles

## 1. Shared Life First

Every feature should benefit both Shaun and Maria.

If a feature is personal, it belongs in another project.

---

## 2. One Source of Truth

Information should only exist in one place.

Example:

Wedding Planning.xlsx

↓

ShaunMariaOS

↓

Telegram

---

## 3. Solve Real Problems

Every feature must save time or reduce mental load.

No unnecessary features.

---

## 4. Keep Modules Independent

Each module should focus on one responsibility.

Examples:

Calendar

Wedding

Finance

Home

Documents

---

## 5. Build for the Future

Write code that is easy to understand, maintain and extend.

---

## 6. Automation Before Repetition

If something is repeated often, automate it.

---

## 7. Simplicity Wins

Choose the simplest solution that solves the problem well.

---

## 8. Preserve Memories

ShaunMariaOS should not only organise life.

It should preserve life's journey.

Wedding.

Home.

Milestones.

Documents.

Anniversaries.

---

## 9. Security Matters

Personal information stays private.

Sensitive files are never committed to GitHub.

Secrets belong in:

.env

credentials.json

token.json

---

## 10. Grow With Us

ShaunMariaOS is designed to evolve alongside Shaun & Maria.

From engagement...

To marriage...

To home ownership...

To every chapter that follows.
